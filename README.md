# 🌱 TIL (Today I Learned)

公式ドキュメントの読み解きや技術ブログの試行錯誤、そして日々の開発での気づき。それらを通して得た学びを、未来の自分や同じ悩みを持つ人のために言語化して記録しています。

---

## 🛠 運用ポリシー (Policies)

「継続」と「環境の健全性」を大切にするための、3つのマイルールです。

1. **PC（ホストOS）を汚さない**
   - ツールやプログラミング言語を Mac にそのまま直接入れず、`asdf` や `rbenv`、`nodenv` など、プロジェクトごとにバージョンを切り替えられる仕組みを使う手順を優先してメモします。OSそのものが不安定になるのを防ぐためです。
2. **「なぜそうしたか（Context）」を添える**
   - 単なる「やり方」のコピペではなく、「その設定を選んだ理由」や「その時に自分がどう考えたか」を一行添えるようにします。
   - **「知識がゼロの状態からでも、背景や手順の理由がわかる」**ような記述を心がけます。数ヶ月後の自分を助けるためのメモでもあります。
3. **完成度よりスピード**
   - 最初から完璧な記録を目指すと続かないので、まずは「動いたコマンド」だけでもOKとします。気づいた時に少しずつ修正していきます。
4. **「一次情報（Ground Truth）」へのパスを確保する**
   - 独自の解説だけで終わらせず、必ず根拠となる**公式リファレンスの該当セクション**や、GitHub 上の**ライブラリのソースコード定義**（例: Rails の `ja.yml` マスター）への直リンクを添えるようにします。
   - 情報が古くなっても、そこを辿れば常に「正解」へ戻れるようにするためです。
5. **関心の分離（Why と How の切り分け）**
   - 「なぜそれを選んだのか（設計思想・比較）」と「具体的にどう書くのか（実装手順）」は、必要に応じて柔軟にファイルを分割します。これにより、ドキュメントが「やり方」のノイズに埋もれず、純粋な知見（ナレッジ）として洗練されるようになります。

## 🌲 命名規則とフォルダ構成のイメージ(Naming Rules & Directory Layout)
後から探しやすくするため、フォルダ名は **`[親言語]_[役割]_名称`** というルールで分類します。
※ 下記はルールのイメージを掴むための例です。実際には、学習の進捗に合わせてこれらのフォルダを随時作成していきます。（フォルダ構成そのものは、リポジトリの成長に合わせて変化します）
| 接頭辞 | 意味 / 役割 | 命名の例 |
| :--- | :--- | :--- |
| **Cmn** | Common (共通設定) | `[Cmn]_[Env]_Setup` |
| **JS** | JavaScript | `[JS]_[Run]_Nodejs` |
| **Ruby** | Ruby | `[Ruby]_[Lang]_Basic` |

### 📝 ファイル名の接尾辞ルール (Suffix Rules for Files)

ファイルを開かなくても「その文書の主目的（Why/How/What）」が一目でわかるように、ファイル名の末尾に役割を示す接尾辞を付与します。
一つのファイルに複数の目的が混在している場合は「読者がそれを読んで最終的に何をするのか（Primary Intent）」で強い方に倒します。

| 分類 | 目的 (主たる関心) | 接尾辞の推奨例 | ファイル名の例 |
| :--- | :--- | :--- | :--- |
| **Why** | なぜそれを選ぶのか（設計思想・比較） | `-comparison`, `-strategy`, `-concept` | `role-enum-vs-boolean-comparison.md` |
| **How** | どうやって達成するのか（手順・解決） | `-guide`, `-howto`, `-workflow` | `rails-i18n-localization-guide.md` |
| **What** | それは一体何なのか（仕様解説・仕組み） | `-spec`, `-anatomy`, `-reference` | `autocomplete-attribute-spec.md` |

### 📁 フォルダ構成の具体例 (Examples)

```text
til/
├── [Cmn]_[Env]_Setup/        # PC環境構築。nodenv（言語のバージョン管理）など
├── [Cmn]_[Tool]_Git/         # Git（変更履歴を保存するツール）や設定
│
├── [JS]_[Run]_Nodejs/        # Node.js（OS上でJavaScriptを動かすエンジン）
├── [JS]_[Lib-Pkg]_React/     # React（Web画面の部品を作るための道具箱）
├── [JS]_[Lib-Pkg]_dbdocs/    # dbdocs（データベースの設計図をWebに公開するツール）
│
├── [Ruby]_[Lang]_Basic/      # Ruby（プログラムを書くための言葉）の基本
├── [Ruby]_[Fw]_Rails/        # Rails（Webアプリを効率よく作るための決まり事）
└── [Ruby]_[Lib-Gem]_RSpec/   # RSpec（書いたコードが正しく動くかテストする道具）
```
