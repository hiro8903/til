# Node.js 環境構築ガイド (nodenv導入編)

このドキュメントは、macOS環境において「PCを汚さない」ポリシーに基づき、`nodenv` を使って最適な **Node.js** 環境を整えるための手順書です。

---

## 💻 検証環境 (Environment)

この手順は、以下の環境で動作を確認しています。

- **OS**: macOS Sequoia 15.4.1 (Apple Silicon / M2)
- **Shell**: zsh
- **Package Manager**: Homebrew

---

## 💡 この作業で「何をしているのか？」の全体像

初心者が最も混乱するのは、「何をインストールしているのか？」という点です。
ここでは、それぞれのツールの役割を「道具箱」に例えて整理します。

| ツール名 | 役割 / イメージ | いつ手に入るか |
| :--- | :--- | :--- |
| **Node.js** | **「エンジン本体」** です。プログラムを動かします。 | このガイドの手順で導入 |
| **nodenv** | **「管理ボックス」** です。複数のバージョンを切り替えます。 | このガイドの手順で導入 |
| **npm** | **「道具箱」** です。便利なツールを管理します。 | Node.jsを入れると **自動で付いてくる** |
| **npx** | **「道具を直接使う機能」** です。 | npmの中に **最初から入っている** |

この作業は、**「管理ボックス(nodenv)を設置し、その中にエンジン(Node.js)を取り付ける」** という二段階の作業を行っています。

---

## 1. 管理ツール（nodenv）のインストール
まずは、Node.jsのバージョンを管理する「土台」をインストールします。

* **目的**: システム全体を汚さず、プロジェクトごとにNode.jsのバージョンを切り替えられる状態にする。

* **コマンド**:
  ```bash
  # Homebrewを使用して管理ツールを導入
  brew install nodenv node-build
  ```

* **それぞれの役割（ここが重要です！）**:
  * **nodenv**: **「管理ボックス（切り替えスイッチ）」** です。現在どのバージョンを使っているかを管理します。
  * **node-build**: **「Node.js のインストール用レシピ集（道具）」** です。`nodenv` に「新しい Node.js をダウンロードして組み立てる機能」を持たせます。これがないと `nodenv install` という命令が使えません。

> [!NOTE]
> **「○○env シリーズ」とレシピ本の関係**
> `nodenv` は元々 Ruby 用の `rbenv` を元に設計されたため、仕組みがそっくりです。しかし、中身の「レシピ本」が異なります。
> * **node-build**: **Node.js** を作るためのレシピ本。（今回使うもの）
> * **ruby-build**: **Ruby** を作るためのレシピ本。（Rails 開発などで使うもの）
> 
> これらはあくまで **「手順書（レシピ本）」** をインストールするだけであり、**これを入れただけでは Node.js や Ruby 本体のインストールは行われません。** 本体のインストールは、この後の手順（`nodenv install ...`）で行います。

### 💡 トラブルシューティング：`Permission denied` が出た場合
もし `brew install` の途中でエラー（Permission denied @ rb_sysopen ...）が出た場合は、Mac の権限設定が原因です。以下のコマンドで修理（所有権の修正）を試みてください。

```bash
# 所有権を自分 (whoami) に戻して正常化する呪文
sudo chown -R $(whoami) /opt/homebrew/Cellar/node-build
brew link node-build
```

---

## 2. エンジン（Node.js）の導入と有効化
実際にプログラムを動かすための本体（Node.js本体）を用意します。

* **目的**: 特定のバージョン（例: 20.11.1）をプロジェクト専用で使用可能にする。

* **コマンド**:
  ```bash
  # 1. 必要なバージョンをインストール
  # ※完了まで数分〜10分程度かかる場合があります。
  nodenv install 20.11.1

  # 2. 実行ディレクトリ（プロジェクトフォルダ）へ移動
  # ※重要：local設定を行う際、必ずそのプロジェクトのフォルダ内で実行してください
  cd ~/path/to/your-project

  # 3. このディレクトリで使用するバージョンを固定
  # 実行後、このフォルダ内に「.node-version」というファイルが自動生成されます
  nodenv local 20.11.1

  # 4. コマンドのパスを最新の状態に更新（おまじない）
  # 新しいバージョンを入れた後は、これを打つことで認識されるようになります
  nodenv rehash
  ```

---

## 3. 設定の反映（シェルへのパス通し）
Macのシェル（zsh）に nodenv を認識させ、どのフォルダにいても Node.js が使えるようにします。

* **目的**: ターミナルの設定ファイル（.zshrc）に「nodenvを起動する」という命令を書き込み、有効化する。

* **手順**:
  1. 設定ファイル（`~/.zshrc`）をテキストエディタで開き、末尾に以下の設定を追記します。
  ```bash
  # nodenv: Node.jsのバージョン管理を自動初期化
  export PATH="$HOME/.nodenv/bin:$PATH"
  eval "$(nodenv init -)"
  ```

  2. 設定を現在のターミナルに即時反映させます。
  ```bash
  source ~/.zshrc
  ```

  3. 疎通確認（バージョン番号が表示されればOK）
  ```bash
  node -v
  ```

> [!IMPORTANT]
> **設定ファイルの場所について**
> この手順書では `~/.zshrc` への直接追記を基本としていますが、設定ファイルを `~/dotfiles` フォルダ等で一括管理している場合は、適宜追記場所を読み替えてください。

### トラブルシューティング：設定が反映されない場合
もし `source ~/.zshrc` を実行しても `node -v` 等でエラーが出る場合は、**「設定ファイルとホームディレクトリの繋がり」**が切れている可能性があります。特にお手製 `dotfiles` 管理の場合は、以下のコマンドでリンクを再作成してみてください。

```bash
# 「ドットファイル管理用フォルダ」と「Macの正規の場所」を紐付ける
ln -sf ~/dotfiles/.zshrc ~/.zshrc
```
