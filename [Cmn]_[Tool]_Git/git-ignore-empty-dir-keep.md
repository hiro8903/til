# 📜 [Cmn]_[Tool]_Git (空フォルダが消える問題と `.keep` の賢い使い方)

GitHub Actions で CI（自動テスト）を動かした際、`cannot load such file -- ... /test/system (LoadError)` というエラーが発生した原因と、その解決策についての備忘録です。

---

## 🌟 結論 (Fast Workflow)

Git は **「中身が何もない空っぽのフォルダは無視する（アップロードしない）」** という仕様を持っています。

**【解決策】**
空のフォルダの中に、**`.keep`**（または `.gitkeep`）という名前の「空の隠しファイル」を1つだけ作成してコミットする。これだけで Git がフォルダを認識してくれるようになります。

---

## 🛠️ なぜ起きたのか？ (Anatomy / 原因の解剖)

1. **ローカルには存在していた**: `rails new` を実行した際、手元のPCには自動テスト用の `test/system/` フォルダが作成されていました。
2. **Git には無視された**: まだシステムテストのコードを1つも書いていなかったため、このフォルダは完全に「空っぽ」でした。Git の仕様上、空フォルダは追跡（Track）されません。
3. **GitHub 上で消滅した**: フォルダが無視されたまま GitHub へプッシュされたため、GitHub 上のコードには `test/system/` フォルダが存在しませんでした。
4. **CI がパニックを起こした**: GitHub Actions（CI）が `bin/rails test:system`（機能テストの実行コマンド）を叩いた時、「テスト用のフォルダ（`test/system/`）がない！読み込めない！」と判断し、`LoadError` でクラッシュしてしまいました。

---

## 💡 なぜこの慣習が生まれたのか？公式見解とコミュニティの対応

この問題はバグではなく、Gitの根本的な設計思想（仕様）から来ています。

### 1. Gitの公式見解（仕様としての制限）
Gitは「コンテンツ（ファイルそのもの）」をスナップショットとして追跡するよう設計されており、「コンテナ（フォルダ）」を追跡する仕組みは意図的に排除されています。

*   **旧公式FAQの回答**: 「Gitの設計上、空のディレクトリを追加することは不可能です。（中略）回避策として、追跡したいディレクトリの中にファイルを入れておく方法があります。」
*   **参考リンク**: [Git FAQ (Can I add empty directories?)](https://git.wiki.kernel.org/index.php/Git_FAQ#Can_I_add_empty_directories.3F)
*   **現行公式FAQの回答**: 該当の記載が見当たりませんでした。
*   **参考リンク**: [Git Reference - Common Issues](https://git-scm.com/docs/gitfaq#_common_issues)

### 2. コミュニティの「非公式な裏技」
Gitが公式にサポートしないため、世界中のエンジニアは**「とりあえず中空のファイルを1つ置いておく」**という回避策（Workaround）を生み出しました。
そこでよく使われるファイル名が `.gitkeep` です。これはGitの公式機能ではなく、ただの「開発者同士の暗黙の了解・おまじない」にすぎません。

### 3. Rails 公式の対応（フレームワークとしての標準化）
Ruby on Rails の公式チームは、この「Gitの仕様による無用なクラッシュ」を防ぐため、フレームワーク自体の標準生成プロセスとしてこの裏技を組み込んでいます。
`rails new` でアプリを作った時点から、`log/` や `tmp/` など「Webアプリの実行時に絶対に必要な空フォルダ」の中には、**自動的に `.keep` というダミーファイルが生成される**ようにプログラミングされています。

*   **Railsの選択**: コミュニティで一般的な `.gitkeep` という名前を使わず、よりシンプルで汎用的な `.keep` という名前を意図的に採用しています。

---

## 🛠️ コマンド例 (CLI)
ターミナルで該当のフォルダに行き、以下のコマンドを打つだけで手動で解決します。

```bash
# test/system フォルダの中に .keep を作成する
touch test/system/.keep

# Git に覚えさせてコミットする
git add test/system/.keep
git commit -m "fix: add .keep to track empty directory"
```
