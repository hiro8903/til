# 👮‍♂️ RuboCop の基本と自動修正 (`-a` vs `-A`)

## 📖 概要
**RuboCop（ルボコップ）**は、Rubyにおける「コードの風紀委員（静的コード解析ツール・リンター）」です。
コードのインデント（空白）のズレから、Rubyらしい美しい書き方の作法までを自動でチェックしてくれます。

### 🍣 Rails 8 と `rubocop-rails-omakase`
これまでRuboCopは「使いたい人が自分で入れる外部ツール」でしたが、**Rails 8 からは事実上の標準機能として組み込まれています。**

プロジェクト作成時に自動で入る `rubocop-rails-omakase` というルールセットには、「細かなルールの議論はRailsコアチーム（大将）に**お任せ**して、開発者は美味しいアプリを作ることに集中しなさい」という哲学が込められています。

公式リポジトリ: [rubocop-rails-omakase (GitHub)](https://github.com/rails/rubocop-rails-omakase)

## 🛠️ 自動修正コマンドの違い (`-a` と `-A`)

CI（GitHubのテスト等）でRuboCopに怒られた場合、手作業で1つずつ直す必要はありません。コマンド一発で自動修正（Auto-correct）させるのが基本です。

### 🟢 `bin/rubocop -a` (推奨: 安全第一)
*   **挙動**: `Safe auto-correct` (安全な自動修正)
*   **直すもの**: スペースの数、改行、クォーテーションの種類など、**「見栄え（フォーマット）」だけ**を直します。
*   **Context（なぜこれを使うか）**: 処理のロジック（プログラムの意味）は全く変わらないため、バグを生む心配がありません。**日常的な修正には必ず小文字の `-a` を使います。**

### 🔴 `bin/rubocop -A` (注意: 豪腕・ベテラン向け)
*   **挙動**: `Unsafe auto-correct` (非安全・破壊的な自動修正を含む)
*   **直すもの**: `-a` の内容に加え、「無駄な `if` 文を短くする」「古いメソッドを新しい高速なメソッドに書き換える」など、**コードの実行内容自体（ロジック）**にまで手を入れてきます。
*   **Context（なぜ注意が必要か）**: プログラムの意味合いが変わってしまう可能性があるため、修正後にテスト（RSpecなど）が通るかを入念に確認する必要があります。

---

## 💻 具体的な使い方

```bash
# プロジェクト全体を「安全に」自動修正する
$ bin/rubocop -a

# 今自分が編集したファイルだけを重点的に直す場合
$ bin/rubocop -a app/models/user.rb
```

## 📝 現場の暗黙のルール・Tips
*   **「コミット前に `-a`」**: 開発現場では、書いたコードを `git commit` する前に必ず `bin/rubocop -a` を走らせ、フォーマットを綺麗にしてからチームに共有（PR）するのがマナー（GitHub Flow）とされています。
*   **CIを塞き止めた時は `fix:` や `style:`**: RuboCopのエラーでCIが赤くなった（止まった）のを直した際のコミットは、単なる見栄えだけであれば `style:`、バグと同じ扱いで直したことを強調するなら `fix:` の接頭語を使うのが綺麗です。
