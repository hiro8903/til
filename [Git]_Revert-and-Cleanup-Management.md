# Git：特定のコミットへ「完全に」戻す手順とクリーンアップ管理

開発環境を特定の状態（過去のコミットなど）に正確に戻したい場合、`git reset` だけでは不十分なケースがあります。このドキュメントでは、不要なファイルを残さずに「さらさらな状態」へ戻すための手順をまとめます。

---

## 1. バックグラウンド・プロセスの停止（ゾンビ現象の防止）

クリーンアップ作業を行う際、まず最初に行うべきは関連プロセスの停止です。

- **原因**: `bin/dev` や `foreman` などの監視プロセスが動いていると、ファイルを消した瞬間に「ファイルが消えた」ことを検知して、親切に再ビルド（復元）してしまいます。
- **対策**: `reset` や `clean` を行う前に、必ず監視プロセスを停止させます。

```bash
# プロセスを止めてからクリーンアップするのが正攻法
pkill -f "bin/dev|foreman|rails" 
```

---

## 2. コミットログの確認とターゲットの決定

戻りたい地点のハッシュ値（ID）を確認します。

```bash
git log --oneline
```
- **ハッシュ値**: 各行の先頭にある `496c0ac` などの英数字です。
- **ポイント**: 戻りたい地点を特定したら、その値をコピーしておきます。

---

## 3. 既存の変更を破棄して戻る（git reset）

Git が追跡している「既存ファイル」の変更を、指定したコミットの状態まで強制的に戻します。

```bash
git reset --hard <ハッシュ値>
```
> [!WARNING]
> **注意**: このコマンドを実行すると、まだコミットしていない現在の変更（trackedなもの）はすべて上書きされ、消去されます。

---

## 4. 追跡外のゴミを掃除する（git clean）

`git reset` を実行しても、新しく作成したファイルや自動生成されたフォルダ（**Untracked** なもの）は、そのまま作業ディレクトリに残ってしまいます。これらを一掃して「さらさら」にするためにクリーンアップを行います。

```bash
git clean -fd
```
- **`-f` (force)**: 強制。削除を確認なしで実行するために必須です。
- **`-d` (directory)**: フォルダも対象。ファイルだけでなく、空フォルダなども含めてすべて削除します。

### 💡 `.gitignore` との関係（重要！）
`git clean` を実行した際、`.env` などのファイルが消えるかどうかは、以下の **「3つのパターン」** のどれに当てはまるかで決まります。

| パターン | ファイルの状態 | 実行コマンド | **結果** |
| :--- | :--- | :--- | :--- |
| **A (安全)** | **`.gitignore` に登録済み** | `git clean -fd` | **削除されません（残る）** |
| **B (注意)** | **`.gitignore` に未登録** | `git clean -fd` | **削除されます** |
| **C (危険)** | **`.gitignore` に登録済み** | `git clean -fdx` | **削除されます** |

> [!IMPORTANT]
> **結論**: `.env` などの重要なファイルを確実に残したい場合は、以下の2点を守る必要があります。
> 1.  そのファイルを必ず **`.gitignore` に登録** しておくこと。
> 2.  `git clean` を実行する際は、**オプションに `x` を含めない** こと。

---

### 💡 補足：`git restore .` と `git reset --hard HEAD` の違い

最新のコミット（HEAD）の状態に戻す際、以下の2つのコマンドは似ていますが、**`git add`（ステージング）後の扱い**が異なります。

| 項目 | `git restore .` | `git reset --hard HEAD` |
| :--- | :--- | :--- |
| **作業中の変更 (Unstaged)** | **すべて破棄** | **すべて破棄** |
| **ステージした変更 (Staged)** | **残る（破棄されない）** | **すべて破棄** |
| **主な用途** | ファイル単位の部分的な書き戻し | 失敗した作業の完全なやり直し |

---

## まとめ：完全にリセットする黄金手順

1. **プロセス停止**: `pkill` または `Ctrl + C` で監視を止める。
2. **ログ確認**: `git log --oneline` で戻りたい地点を探す。
3. **リセット**: `git reset --hard <hash>` で既存ファイルを戻す。
4. **お掃除**: `git clean -fd` で追跡外の残骸（不純物）を消す。
