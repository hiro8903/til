# 🧟‍♂️ ゾンビ現象（勝手に復活するファイル・ディレクトリ）のメカニズムと分類

「手動でファイルを消したはずなのに、数秒後にあるいは再起動時に勝手に復活している」という事象（通称：ゾンビ現象）について、**関心の分離（どのようなコンテキストで起きているか）** ごとに整理・分類します。

ゾンビ現象は「アセットやビルド特有のバグ」というわけではありません。本質的には、**「裏側で自動的にファイルを監視・作成するプロセス」** と **「手動でのファイル操作（Git の復元など）」** が衝突した時に発生する、モダン開発特有の現象です。

これは大きく分けて、以下の2つの「関心」に分類できます。

---

## 🏗️ 分類 A：監視プロセスとの競合（Git 操作時によく発生）
**関心領域:** プロセス管理・状態変更（Git vs File Watcher）

今回 `git restore` や `git reset` 時などに発生した典型的なケースです。

### 🚨 発生シナリオ
1. ターミナルで `bin/dev`（Tailwind や Webpack の監視プロセス）を起動したままにしている。
2. 別のターミナルで `git clean -fd` や `git reset --hard` を実行し、自動生成された CSS 等を「消去」して過去に戻ろう（リセットしよう）とする。
3. **【衝突】**: 監視プロセスが「いま、アセットフォルダの中身がごっそり書き変わった（消された）ぞ！」と変更を素早く検知し、親切心から瞬時に再ビルドしてファイルを元の場所に出力してしまう。
4. **【結果】**: Git で完全に消したはずのファイルが「ゾンビ」のように数秒で復活する。

### 💡 対策（正攻法）
この現象の元凶は「バックグラウンドで生きている監視プロセス」です。Git などの大幅なファイル操作を行う際は、必ず**手作業と自動作業の競合を防ぐ**必要があります。
- **解決策**: `git reset` などのクリーンアップ操作を行う前に、稼働中のプロセス（`bin/dev` や `foreman`）を必ず終了させる（`Ctrl + C` または `pkill -f foreman`）。

> **関連ドキュメント**: [【Git】Revert-and-Cleanup-Management](./[Git]_Revert-and-Cleanup-Management.md)

---

## 🛠️ 分類 B：ツールの初期化仕様（起動時によく発生）
**関心領域:** 環境構築・ツールの仕様（Gem Initializer vs File System）

これは、手動でフォルダを消したのに、サーバーを起動（`rails s` や `bin/dev`）した瞬間に特定のディレクトリ（例：特定の空フォルダなど）が「絶対に」復活してしまうケースです。

### 🚨 発生シナリオ
1. 何らかのエラーを解決するために、原因と思われるディレクトリを手動で消す（`rm -rf app/assets/builds/tailwind/`）。
2. プログラムを再起動する（`bin/dev` を叩く）。
3. **【衝突】**: 対象の Gem（例：Tailwind のインストーラー）が、「このフォルダが無いと機能が破綻するから作ってあげよう」という仕様に沿って、起動スクリプト内で強制的に `mkdir -p` を実行する仕組みを持っている。
4. **【結果】**: 起動のたびにゾンビのようにフォルダが生成され続け、他の仕組み（Propshaft のディレクトリ判定など）と競合してエラーを引き起こし続ける。

### 💡 対策（正攻法）
この元凶はプロセスではなく「ツールの仕様（ハードコードされた命令）」にあるため、プロセスを停止させても解決しません。
- **解決策**: プログラムの仕様であると受け入れ、パスの競合によるエラーの場合は、フォルダを消すこと以外のアプローチ（設定ファイルで出力先を変える、該当ツールのアップデートを待つなど）を検討する。

> **関連ドキュメント**: [Rails 8 アセットビルドの裏側](./[Ruby]_[Fw]_Rails-Frontend-Standard/rails-asset-build-mechanics.md)

---

## ⚖️ まとめ：ゾンビへの処方箋
ゾンビ現象に遭遇した時は、以下の2つのどちらの「関心」から発生したかを切り分けるのがプロの手順です。

1. **「作業中（Git編）にすぐ蘇るゾンビ」**
   - 犯人：監視プロセス（Foreman 等）
   - 対策：**「プロセスを止めろ！」**
2. **「起動時（環境構築時）に必ず蘇るゾンビ」**
   - 犯人：ライブラリの初期化コード
   - 対策：**「仕様を疑い、設定を見直せ！」**
