# 🌍 Rails の `i18n`（国際化・日本語化）の基礎とよく使われる実践例

🔗 **公式ドキュメント**:
*   [Rails Guides: Rails Internationalization (I18n) API](https://guides.rubyonrails.org/i18n.html)
*   [rails-i18n/ja.yml (GitHub)](https://github.com/svenfuchs/rails-i18n/blob/master/rails/locale/ja.yml)
    *   ※ Rails標準のバリデーションメッセージや日付形式など、コミュニティによってメンテナンスされている「日本語の公式デフォルト設定」の定義ファイルです。

## 📖 概要
Rails には、あらゆる英語のシステム名（モデル名やカラム名、ステータス等）を、コードを汚さずに美しい母国語（日本語）に変換して画面表示するための仕組み **「i18n (Internationalizationの略: `i` と `n` の間に18文字あるため)」** が備わっています。

*   🗣️ **読み方**: 「アイ・エイティーン・エヌ」と読みます。現場ではそのまま「あいじゅうはちえぬ」と呼ぶ人も非常に多いです。

すべては基本的に、`config/locales/ja.yml` という辞書ファイルに「英語キー」と「日本語訳」をマッピングしていくことで実現します。

以下に、開発現場で真っ先に必要になる「代表的な i18n の使い方 4選」を整理しました。

---

## 📚 1. カラム名・モデル名の日本語化 (ActiveRecord)

DB のテーブル名やカラム名をそのまま画面に出すと `last_name` のようになって不恰好です。これらを自動で「姓」などに変換させる一番基本の使い方です。

### `ja.yml` の定義
```yaml
ja:
  activerecord:
    models:
      user: "ユーザー"
      facility: "拠点"
    attributes:
      user:
        user_code: "ユーザーコード"
        name: "氏名"
        password: "パスワード"
```

### ビューでの呼び出し
フォームのラベルなどで自動的に日本語が適用されます。（バリデーションの「パスワードを入力してください」などのエラー文の主語も、すべてこの名前が自動で使われます！）
```erb
<%= User.model_name.human %>               #=> "ユーザー"
<%= User.human_attribute_name(:name) %>    #=> "氏名"
```

## <div id="enum-i18n">🎨 2. `enum`（列挙型）ステータスの日本語化</div>

DBの「数値 (`0`, `1`)」やコード上の「英語シンボル (`:admin`)」を、画面専用の日本語（管理者）に変換するテクニックです。

### モデルの定義 (`app/models/user.rb`)
```ruby
class User < ApplicationRecord
  enum :role, { field_worker: 0, admin: 1 }
end
```

### `ja.yml` の定義
`activerecord.enums.[モデル名].[カラム名]` の階層にマッピングを作ります。
```yaml
ja:
  activerecord:
    enums:
      user:
        role:
          field_worker: "一般作業員"
          admin: "システム管理者"
```

### ビューでの呼び出し
`[カラム名]_i18n` という魔法のサフィックス付きメソッドが自動生成されるので、それを呼び出すだけです。
```erb
<%= @user.role_i18n %> #=> "システム管理者" と表示される
```

### 💡 なぜ Enum だけ特定のメソッドが必要なの？（「項目名」と「中身」の違い）

勘違いしやすいポイントですが、Rails の自動変換には「自動でいけるもの」と「明示が必要なもの」の 2 種類があります。

1.  **項目名 (Label)**:
    *   `last_name` を「姓」という見出しに変えるのは「自動」です。Rails は画面を作る際「この項目の名前(見出し)は何？」と辞書を勝手に見に行ってくれます。
2.  **項目の中身 (Value)**:
    *   Enum の値（`0` や `:admin`）を「システム管理者」という文字列に変えるのは「明示的」な指示が必要です。
    *   なぜなら、`0` という文字をそのまま出したいのか、日本語に直したいのか、それともアイコンを出したいのかは、開発者がその都度決めるべき「データ」だからです。

そのため、Enum の中身を日本語で出したい時は、`@user.role_i18n` のような魔法のサフィックス（※ `enum_help` 等の Gem を併用するのが一般的です）を使って、明示的に「翻訳して！」と命令する必要があります。

## 💬 3. 画面の静的テキスト（ボタンや見出し）の日本語化

「保存する」や「次へ」といった画面に直接書くテキストも、将来の言語追加（英語対応など）に備えて i18n を使うのがベストプラクティスです。

### `ja.yml` の定義
特別な階層縛りはなく、好きな名前（ディレクトリごとの整理など）で自由に定義できます。
```yaml
ja:
  views:
    buttons:
      save: "保存する"
      cancel: "キャンセル"
    users:
      index:
        title: "ユーザー一覧"
```

### ビューでの呼び出し
`t` （`translate`の略）メソッドに辞書のキーパスを渡して呼び出します。
```erb
<h1><%= t('views.users.index.title') %></h1> #=> "ユーザー一覧"
<%= button_tag t('views.buttons.save') %>    #=> "保存する"
```

## 🚨 4. フラッシュメッセージ（通知）の日本語化

コントローラーで設定する「保存しました」といった通知も辞書管理します。

### `ja.yml` の定義
```yaml
ja:
  flash:
    users:
      create:
        success: "ユーザーの作成に成功しました。"
      destroy:
        alert: "ユーザーを削除しました。"
```

### コントローラーでの呼び出し
コントローラー内でも `t` メソッドが使えます。
```ruby
def create
  # ... 保存処理 ...
  redirect_to @user, notice: t('flash.users.create.success')
end
```

---
## ✅ まとめ
システムの文言（日本語）を **「コードの中（RubyやHTMLファイル）に直書きしないこと」**。これが保守性の高いクリーンなアプリケーションへの第一歩です！すべての文字は `ja.yml` が握っている状態を目指しましょう。
