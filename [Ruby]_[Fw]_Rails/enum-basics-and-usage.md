# 🧙‍♂️ Rails `enum` (列挙型) の基本と魔法のメソッドたち

🔗 **公式ドキュメント**: [Rails API: ActiveRecord::Enum](https://api.rubyonrails.org/classes/ActiveRecord/Enum.html)

## 📖 概要
Rails の **`enum` (エナム/列挙型)** は、データベース上の「数値 (0, 1, 2...)」に対し、プログラム上で「意味のある名前 (:admin, :staff...)」を割り当てて管理するための強力な機能です。

単に名前を付けるだけでなく、Rails が裏側で**「魔法のような便利なメソッド」**を大量に自動生成してくれます。

---

## 🛠️ 1. 基本的な定義方法

モデルファイル（例: `user.rb`）で以下のように記述します。

```ruby
class User < ApplicationRecord
  # DB上のroleカラム（integer型）の値を、以下のように解釈する
  enum :role, {
    field_worker: 0, # 一般作業員
    admin: 1,        # システム管理者
    manager: 2       # 工場長
  }
end
```

### 📋 なぜ DB は Integer (数値) なの？
*   **高速性能**: データベースにとって、長い文字列より数値を比較する方が圧倒的に速く、データ容量も節約できます。
*   **整合性**: プログラム側で `admin` を `administrator` に書き換えたとしても、DBの中身（数値の1）を変える必要がないため、安全です。

---

## ✨ 2. `enum` が生み出す「魔法のメソッド」

`enum` を定義した瞬間、Rails は以下のメソッドを自動的に生み出します。

### 🔍 状態を調べる (❓)
わざわざ `user.role == "admin"` と書く必要はありません。
```ruby
user.admin?         #=> role が 1 なら true
user.field_worker?  #=> role が 0 なら true
```

### 🆙 状態を更新する (❗️)
わざわざ `user.update(role: 1)` と書く必要はありません。
```ruby
user.admin!         #=> role を即座に 1 (admin) に書き換えて保存する
user.manager!       #=> role を即座に 2 (manager) に書き換えて保存する
```

### 🎯 検索する (Scopes)
クラス全体に対して、特定の役割のユーザーだけを簡単に抽出できます。
```ruby
User.admin          #=> 全ての「システム管理者」を抽出する
User.not_admin      #=> 「システム管理者以外」を抽出する
```

---

## ⚠️ 現場で絶対に守るべき 1 つのルール

**「定義の途中に新しい項目を割り込ませたり、数値をずらしたりしてはいけない」**

```ruby
# ❌ ダメな例：途中にリーダーを入れて、以降の数値をずらす
enum :role, {
  field_worker: 0,
  leader: 1,  # ここに割り込ませて、adminを2にずらした！
  admin: 2,
  manager: 3
}
```
これをやってしまうと、**既にDBに保存されている「1」という値を持つユーザーが、全員「管理者」から「リーダー」へ勝手に降格されてしまう**という大事故が発生します。新しい項目は、必ず最後に追加するか、空いている数値を明示的に指定して追加するのが鉄則です。

---

## ✅ まとめ
`enum` は、**「DBには軽く、人間には読みやすく」** を実現するための Rails の素晴らしい知恵です。
一度定義すれば、`if user.admin?` のような、まるで英語の文章を読んでいるかのような美しいコードが書けるようになります。
