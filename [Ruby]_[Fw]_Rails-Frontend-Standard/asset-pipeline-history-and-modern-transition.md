# Rails アセット管理の変遷：Sprockets から Propshaft への転換

Rails における CSS や JavaScript の管理（アセットパイプライン）は、Rails 7〜8 を境に大きな歴史的な転換を迎えました。かつての「常識」が、現代の「非常識」へと変わった背景を解説します。

---

## 1. 過去の常識：Sprockets 時代（Bundling）
Rails 3.1 から長らく標準だった `Sprockets` という仕組みでは、**「すべてのアセットを 1 つに合体させて配信する（Bundling）」** ことが絶対の正解でした。

- **読み込み方法**: `<%= stylesheet_link_tag "application" %>`（1つのタグのみ）
- **仕組み**: `application.css` という「目次ファイル（マニフェスト）」の中で、`//= require` という命令を使い、世界中のスタイルシートを無理やり 1 つの巨大なファイルに結合していました。

### なぜ 1 つにまとめる必要があったのか？
-   **HTTP/1.1 の制約**: かつての通信規格では、ファイルを 1 つ読み込むごとに「接続」を確立する必要があり、そのオーバーヘッドが非常に大きかったためです。ファイルが 10 個あれば、10 回の通信が発生し、サイトが劇的に重くなりました。
-   **結論**: 「通信回数を減らすために、巨大な 1 つのファイルにして送る」のが、当時の賢い戦い方（正攻法）でした。

---

## 2. 現代の潮流：HTTP/2 以降（Unbundling）
現代のネットワーク環境では、HTTP/2 という新しい通信規格が普及し、これまでの「合体させる」という戦略が不要になりました。

- **HTTP/2 の特性**: **1 回の接続で、大量の小さなファイルを並列に、かつ高速に読み込むことができます。**
- **Unbundling（非合体）への移行**: 無理に巨大なファイルを作る必要がなくなり、むしろ「役割ごとにファイルを分割したまま配信する」方がメリットが大きくなりました。

### 分割配信（Unbundling）のメリット
1.  **キャッシュ効率**: `tailwind.css`（外部ライブラリ）は変更が少ないですが、`application.css`（自作スタイル）は頻繁に書き換わります。分割しておけば、自作スタイルを直しても、大きな外部ライブラリのキャッシュが有効なままになり、ユーザーの体感速度が上がります。
2.  **ビルド速度**: 巨大な 1 つのファイルを作るための「合体作業」が不要になるため、開発中の待ち時間が短縮されます。

---

## 3. Rails 8 + Propshaft の新しい設計思想
Rails 8 が採用した `Propshaft` は、この「HTTP/2 時代の Unbundling」に基づいた設計になっています。

### `:app` ショートカットの真実
Rails 8 において `<%= stylesheet_link_tag :app %>` と書いた時、裏側では以下のことが起きています。

1.  **自動探索**: ロードパス内をスキャンし、`application.css` と `tailwind.css` の両方を見つける。
2.  **複数リンク生成**: 2 つの `<link rel="stylesheet">` タグを、自動的に別々に書き出す。

これは「不具合で 2 つに分かれた」のではなく、**「現代の通信環境において、最も効率的でメンテナンスしやすい形として、意図的に 2 つに分けて出力されている」** のが正解です。

---

## 4. まとめ
| 項目 | Sprockets 時代（旧） | Propshaft 時代（新） |
| :--- | :--- | :--- |
| **戦略** | Bundling（合体・統合） | Unbundling（分割・並列） |
| **ファイル数** | 極力 1 つにまとめる | 役割ごとに適切に分ける |
| **読み込みタグ** | `<%= stylesheet_link_tag "application" %>` | **`<%= stylesheet_link_tag :app %>`（複数タグを展開）** |
| **背景** | HTTP/1.1（接続が重い） | HTTP/2 / HTTP/3（並列に強い） |

今回のトラブルで「タグが複数出てきて驚いた」のは、まさにこの **「1 つにまとめるべき」という古い呪縛から Rails が解き放たれ、新しい設計思想へ移行した最前線** を目撃したからに他なりません。
