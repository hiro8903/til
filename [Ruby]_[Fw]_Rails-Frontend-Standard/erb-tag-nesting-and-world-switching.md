# ERB の掟：入れ子（マトリョーシカ）禁止と世界の切り替え

## 🕯️ 背景 (Background)
Rails の View（`.html.erb`）では、Ruby のコードと HTML の地文を自由に行き来できるが、その記述には厳格な優先順位と「入れ子」に関する禁止事項がある。

---

## 🚫 1. 禁じ手：窓の中に窓は作れない (No Nested ERB Tags)
ERB の出力タグ `<%= ... %>` の中で、さらに別の `<%= ... %>` を書くことは **シンタックスエラー** になる。

### ❌ 動かない例
```erb
<%= render ..., actions: capture { <%= link_to ... %> } %>
```
- **理由**: コンピュータは `<%=` を見つけた瞬間、次の `%>` が来るまでを「一つの Ruby の世界」とみなす。その中に入れ子のタグがあると、パース（解析）が壊れてしまう。

---

## 🚪 2. 解決策：一旦「地（HTML）」に降りる (Stepping out to HTML)
複雑な HTML や複数の要素を「一つの塊」として扱いたい場合、**`%>` で一度 Ruby の窓を閉じ、HTML の世界（地の文）へ「脱出」する** テクニックを使う。

### ✅ 正しい例
```erb
<%= render ..., actions: capture { %>  <%# 🚪 ここで一旦 Ruby を終了 %>
  <%= link_to "ボタン1", ... %>         <%# 🌍 地の文（HTML）の上で新しい窓を開く %>
  <%= link_to "ボタン2", ... %>
<% } %>                                <%# 🚪 再び Ruby に戻ってカッコを閉じる %>
```

### この構造のポイント
1.  **`{ %>`**: Ruby のカッコ `{` を書いた直後に、**「中身は HTML で書くぞ」** と宣言して窓を閉じる。
2.  **`<% }`**: HTML を書き終えたら、**「Ruby に戻ってカッコを閉じるぞ」** と宣言して窓を再開する。

---

## 🎯 3. なぜこの技術が必要なのか？
1.  **可読性の向上**: 全てを Ruby の文字列として連結するよりも、生の HTML として記述する方が Tailwind クラスの管理やインデントが容易になる。
2.  **`capture` との連携**: `capture` メソッドと組み合わせることで、「HTML の塊」を変数（小包）として別の部品へ安全に届けることができる。

---

## ⚖️ まとめ
- **「窓（`<%=`）」の中に窓は作れない**。
- 複雑な中身を渡したい時は、**一旦外に出て、改めて小さな窓を作る**。
- この「世界の行き来」を使いこなすことで、View の表現力と保守性が劇的に向上する。
