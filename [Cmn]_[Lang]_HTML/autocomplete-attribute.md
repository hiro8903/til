# 🌐 HTML5 の `autocomplete` 属性の仕様と正しい使い方

🔗 **公式ドキュメント**: [HTML Living Standard - The autocomplete attribute](https://html.spec.whatwg.org/multipage/form-control-infrastructure.html#autofill)

## 📖 概要
Rails などのフォームでよく見かける `autocomplete: "username"` は、フレームワークの機能ではなく、ブラウザ（HTML5）の仕様です。
「このフィールドに何のデータが入るか」をブラウザに伝えるための **「ヒントタグ（Hint Attribute）」** として機能します。

---

## 🛠️ `autocomplete` の役割
このヒントがないと、ブラウザは「入力欄の名前」だけを頼りに過去の履歴を推測します。
これを設定することで、以前入力した値の「自動補完（オートコンプリート）」や「1PasswordやiCloudキーチェーンへの保存」がスムーズかつ正確に行われます。

### 📋 設定できる値の種類（固定値抜粋）
HTML仕様で定められた「固定の単語」しかブラウザは理解できません。

| 値 | 意味 |
| :--- | :--- |
| `"username"` | ユーザー名・ログインID全般（メールアドレスでなくてもOK） |
| `"email"` | メールアドレス専用 |
| `"current-password"` | 現在のパスワード（ログイン時） |
| `"new-password"` | 新しいパスワード（パスワード登録・変更時） |
| `"name"` | フルネーム（氏名） |
| `"off"` | 自動補完を無効にする（セキュリティ上保存させたくないワンタイムパスなどに使用） |

---

## ⚠️ 現場での注意点（バックエンドの命名と混同しない）

`autocomplete` で指定する値は、あくまで **「ブラウザ（HTML5仕様）が予め用意している辞書のキーワード」** です。
バックエンド（DBのカラム名や、フロントエンドの変数名）とは完全に切り離して考える必要があります。プログラム側の名前に無理に合わせようと独自の値を指定してしまうと、ブラウザが意味を理解できず、自動補完の恩恵が一切受けられなくなってしまいます。

### ❌ よくある勘違い（独自の名前にしてしまう）
*   **電話番号**: バックエンドの変数に合わせて `autocomplete: "mobile_number"` などと指定する。（正解は仕様にある `"tel"`）
*   **郵便番号**: DBのカラム名の通り `autocomplete: "zip_code"` と指定する。（正解は仕様にある `"postal-code"`）

### 💡 具体例: 独自のログインID（ユーザーコードや社員番号）の場合

特殊な業務システムなどで、ログイン用のIDが `email` でも単純な `name` でもなく、`user_code`（ユーザーコード）という独自の呼称だったとします。

> **❌【勘違いケース】**: 「うちは `user_code` というカラム名だから、`autocomplete: "user_code"` に直さなきゃ！」

**結論から言うと、これは間違いです。**
`user_code` はアプリ固有の呼び方であり、役割としては「ログインに使う識別用のアカウント名」を指しています。そのため、HTML仕様上用意されている `"username"` を指定するのが最も正確な意味を伝えます。

「ログインするためのID項目（メールアドレス、社員番号、学生番号、ユーザーコード等など）」であれば、名前が何であれ、一律でおとなしく `"username"` を指定するのが正しい（パスワードマネージャーと連動させる）正解となります。
