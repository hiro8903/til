# [Ruby] [Tool] Bundler: Gem のバージョン指定（バージョニング）完全ガイド (-spec)

Gemfile における Gem のバージョン指定方法（バージョニング）は、アプリケーションの安定性と保守性に直結する重要な知識である。

---

## 💡 Context (背景)

論理削除用の「discard」gem を導入する際、`gem "discard", "~> 1.3"` と記述したところ、実際には `1.4.0` がインストールされた。
この「~>（チルダ・グレーターザン）」の挙動や、その他の指定方法による意図の違いを整理し、将来の自分や他の開発者が迷わないようにするためにこのドキュメントを作成した。

---

## 🏗 セマンティックバージョニング (Semantic Versioning) の基礎

RubyGems を含む多くのライブラリは、**「メジャー.マイナー.パッチ」** という 3 桁の数字でバージョンを表現する（例: `1.4.0`）。

| 桁 | 名称 | 変化するタイミング | 開発者の意図 |
| :--- | :--- | :--- | :--- |
| 1 桁目 | **メジャー** (Major) | 破壊的変更（互換性がない） | **「別物になったので注意してください」** |
| 2 桁目 | **マイナー** (Minor) | 機能追加（互換性はある） | **「便利な機能が増えたのでアップデートを推奨します」** |
| 3 桁目 | **パッチ** (Patch) | バグ修正のみ（互換性はある） | **「安全に直したので全員当ててください」** |

---

## 🌲 基本的な比較演算子 (Basic Constraints)

特定のバージョン以上、あるいは特定のバージョンそのものを指定したい場合に使用する。

- `gem "discard", "1.4.0"` : **完全一致**。このバージョン以外は認めない、最も厳しい指定。
- `gem "discard", ">= 1.4.0"` : **1.4.0 以上**なら何でも OK。常に最新を追いかけたい場合。
- `gem "discard", "< 2.0.0"` : **2.0.0 より小さい**。破壊的変更を避けたい場合。
- `gem "discard", ">= 1.4.0", "< 1.4.5"` : **複数条件**。「1.4.0以上だが、不具合のある 1.4.5 以降は入れたくない」といったピンポイントな制約。

---

## 🚀 実務で最も多用される記法： `~>` (Pessimistic)

指定の桁数によって、開発者が「どこまで自動更新を許容するか」という**制御の意思表明**ができる。

- **`gem "discard", "~> 1.4.0"` (3 桁指定)**
  - 範囲: `>= 1.4.0` かつ `< 1.5.0`
  - **意図**: 「パッチ（バグ修正）だけ取り込みたい。機能追加（マイナー）で動作が変わるのも怖い」という、**安定性重視**。
- **`gem "discard", "~> 1.4"` (2 桁指定)**
  - 範囲: `>= 1.4.0` かつ `< 2.0.0`
  - **意図**: 「新機能（マイナー）も自動で使いたい。ただし、動かなくなるメジャー変更は止めて」という、**進化と安定のバランス**重視。

---

## 🔗 特殊な場所からの指定 (Git / Path)

公開前の最新コードや、自分のパソコンにあるコードを読み込む。

- `gem "discard", github: "jhawthorn/discard"` : GitHub の最新版（`main` ブランチ）を直接持ってくる。
- `gem "discard", github: "jhawthorn/discard", ref: "a1b2c3d"` : 特定のコミット（一瞬の記録）で完全に固定する。
- `gem "my_custom_gem", path: "../my_custom_gem"` : 自分のパソコンの別フォルダにある自作 Gem を読み込む（開発用）。

---

## 🔧 あまり見かけないが知っておくと役立つ記法

- **プレリリース版の指定（rc, beta, alpha）**
  - `gem "discard", ">= 1.5.0.rc1"`
  - **意図**: 正式リリース前のテスト版（Release Candidate 1）を先行して試したい場合。
- **プラットフォーム限定（Windows / JRuby 等）**
  - `gem "tzinfo-data", platforms: [:mingw, :mswin, :x64_mingw]`
  - **意図**: 特定のOS（例: Windows）の時だけ必要なパーツを読み込ませ、余計なインストールを防ぐ。

## 💡 まとめ：どう書くのが正解か？

基本的には **メジャーバージョンの桁数を固定する `~>` 指定** （例: `~> 1.4`）が、
「セキュリティ更新やバグ修正の恩恵を受けつつ、アプリが壊れるのを防ぐ」というバランスの取れた、**実務上のベストプラクティス** である。
